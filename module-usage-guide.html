VIEWPOINT 1 — Decision Axes

判断軸マップ：5つの軸でモジュールを選ぶ

モジュール一覧から「何を選ぶか」を直接決めず、まず以下の5軸で要件を分解する。軸ごとに対応するモジュール群が決まるため、選択肢を一気に絞り込める。

情報量 1〜2行で済む情報か / 段落単位か / 長文セクションか 1〜2行11リスト／4テキスト 段落4テキスト／5画像＋テキスト 長文8アコーディオン
階層（ページ） そのページはL1（TOP）／L2（インデックス）／L3（テーマ）／L4（個別取り組み）のどれか L11-1 TOPヒーロー L21-2 第2階層ヒーロー L31-3 第3階層ヒーロー L41-4 第4階層ヒーロー
読み方 全員に必読 / 選択的に読む / 参照のみ（用語補足等） 必読直接表示 選択的8アコーディオン 参照9-2 用語モーダル
役割 伝える（強調）／読ませる（本文）／辿らせる（リンク）／開示する（表）／補足する（注釈） 伝える6強調エリア 読ませる4テキスト 辿らせる7リンク 開示する10テーブル 補足する4-3注釈／6-2補足

VIEWPOINT 3 — Adoption Criteria

採用基準：使う条件 / 使わない条件（WHEN / WHEN NOT）

よく使われるモジュールについて、採用条件と非採用条件を1枚にまとめる。クライアント説明・自己レビュー両方で使えるチェック資料。本ガイドv1.0では特に判断要件の多い 8アコーディオン／6-1強調／7-3カード／10-2 KPI表／11-4背景色付きリスト を収録。

8-4 / 8-1 アコーディオン .wf-acc-h3 / .wf-acc-h2
WHEN — こんな時に使う
  • H3配下に同列項目が3つ以上ある（例：TCFDの開示4要素／DE&Iの3テーマ）
  • 各項目が3段落以上の長文セクションになる
  • すべてを必読としなくてよい（選択的に読ませたい）
  • ページ内の構造をコンパクトに見せたい
WHEN NOT — 使わない場面
  • 項目が1〜2個しかない（直接表示で十分）
  • ページの主目的そのものを表す内容（畳むと意図が消える）
  • 既存原文が「短いリード文＋数パラグラフ」しかない（盛らない）
  • 用語1個だけの補足 → モーダル（9-2）に
代替：用語=9-2モーダル／並列見せ=7-3カード／単発本文=直接表示 初期状態：<details open> 必須 階層：H3直下=.wf-acc-h3 / H4直下=.wf-acc-h4
6-1 POINTタグ強調エリア .callout-box
WHEN — こんな時に使う
  • そのページの結論・宣言・主要KPIを引き上げる
  • マテリアリティの中核トピックを示す（DE&I宣言、CARBON MINUS 2025）
  • ページ冒頭で「このページで言いたい1つ」を明示
WHEN NOT — 使わない場面
  • 1ページに3個以上置く（強調が機能しなくなる）
  • 注釈・補足レベルの情報（6-2を使う）
  • 既存原文に該当する強調文が存在しない（盛らない）
タグ文言：ページごとに内容を表す英字短文に（POINT / MATERIALITY KPI / CARBON MINUS 2025 等） NGスタイル：左縦棒型／オーバーレイアイコン型は廃止
7-3 取り組みカード（活動カード） .activity-card
WHEN — こんな時に使う
  • 取り組み／施策／事例をサムネ＋カテゴリ＋要約で誘導
  • L3テーマページ内の代表事例の一覧（2列）
  • L2インデックスから子テーマへの導線（3列）
WHEN NOT — 使わない場面
  • 画像／要約のいずれも揃わない（リンクボタンに）
  • 遷移先がPDFのみ（7-10 link-btn に）
  • カード内画像に「+」拡大ボタンを付ける（NG）
カテゴリタグ：ジャンル別に書く（「取り組み」一律はNG） 列数：004=3列 / 024・025・124・131・148=2列
10-2 KPI 進捗表（多列） .ghg-table
WHEN — こんな時に使う
  • 年度推移・実績・目標を行列で比較
  • 環境・社会データの定量開示
  • 3行以上のデータがある（少ない場合は11リスト or 6-1強調）
WHEN NOT — 使わない場面
  • 行列で比較する必要がない（文章で済む）
  • 規格対照（GRI/SASB）は10-4を使う
  • シナリオ分析（バッジ要）は10-5を使う
最小行数：3行以上（catalog v3.1ルール） 注釈：表下に .term-note で凡例・注釈
11-4 背景色付きリスト .norm-list
WHEN — こんな時に使う
  • 国際規範・規格名・受賞名など「対象群」を強調列挙
  • 箇条として並列しつつ、本文との境界を視覚的に切りたい
WHEN NOT — 使わない場面
  • 通常の箇条書き（11-1 を使う）
  • 順序が重要なリスト（11-2 番号付き）
  • 装飾的に背景色を付けたいだけ（意味のある「対象群」でなければ使わない）
背景色：var(--c-bg-sub)（#F2F2F2）固定 マーカー：◆ 1色のみ（多色化しない）

VIEWPOINT 4 — Page Templates

ページタイプ別 標準構成：そのまま使える組合せパターン

L1〜L4の階層×ページ性質ごとに、「上から順にどのモジュールを並べるか」のテンプレートを示す。クライアントが新ページを企画した際の出発点として使える。

L2 インデックスページ型 適用：004 環境 / 005 社会
1-2第2階層ヒーロー
2-1 + 4-2H2＋導入文
7-5横断インデックスリンク
7-3（3列）取り組みカード一覧
7-4関連リンク

L2は 「テーマ全体を俯瞰させる」 ことが役割。詳細は配下L3に譲り、ここでは横並びの誘導に徹する。カードは3列で網羅性を演出。

L3 テーマ詳細型 適用：024 気候変動 / 124 DE&I / 131 健康経営 / 148 顧客責任・品質
1-3第3階層ヒーロー
13-2右アンカーナビ
6-1POINTタグ強調
2-1 + 8-4H2＋アコーディオン群
7-3（2列）取り組みカード
7-4関連リンク

L3は 「テーマの本文・取り組み・関連先」 の3層構成。長文セクションはアコーディオンに、代表事例はカード2列、外部・関連は最下部にまとめる。アンカーナビで縦長ページのナビゲーション性を確保。

L3 開示・データ型 適用：035 TCFD / 199 ESGデータ / 222 ガイドライン対照表 / 225 イニシアティブ＋社外評価
1-5シンプルヒーロー
13-2右アンカーナビ（任意）
2-1H2セクション
10-2 / 10-4 / 10-5各種テーブル

開示型は 「テーブル中心・装飾は最小限」。POINTタグ強調（6-1）は使わず、データの可読性に振り切る。199 ESGデータは catalog v3.0 ルールでアンカーナビ非設置（凍結事項）。

L3 ポリシー・規程型 適用：232 方針一覧（097）
1-5シンプルヒーロー
2-1H2（規程カテゴリ別）
7-10 .is-full説明文付きリンクボタン × N

規程一覧は 「各規程に等しい重み」。カード化やアコーディオンは使わず、7-10 を .is-full で grid 全幅配置するだけで成立。容量・形式メタを .link-btn-meta で併記。

L3 アーカイブ型 適用：230 サステナビリティサイトアーカイブ（096）
1-5シンプルヒーロー
12-3年度フィルタチップ（任意）
7-10説明文付きリンクボタン（PDF容量メタ付き）

過去PDFの並列。タイトルに 容量併記、.link-btn-meta で発行年・形式を表示。.report-group 系は廃止し、7-10 に統一。

L3 ニュース・一覧型 適用：231 トピックス一覧（105）
1-5シンプルヒーロー
12-3年度フィルタチップ
7-9ニュース・トピックスリスト

時系列の更新情報。1行＝1記事。4行以上を表示（catalog v3.1 ルール）。アンカーナビは非設置（凍結事項）。

L4 個別取り組み型 適用：025 再エネ / 098 健康宣言 / 099 労働安全衛生方針
1-4第4階層ヒーロー
13-2右アンカーナビ
5-1 / 5-2画像＋テキスト
6-1 + 11系強調＋リスト
7-4関連リンク

L4は 「単一施策の説明」。L3より具体的な画像PH・図表が増える。1-4ヒーローはパンくず＋親リンクで「どこから来たか」を明示する。

VIEWPOINT 5 — Cross-cutting Rules

横断ルール：モジュール単独に紐づかない全体規律

カタログには載っていないが、全モジュール／全ページに適用される共通ルール。実装フェーズで意図がズレないよう、ここで明文化する。

既存原文厳守
既存サイトに対応原文・画像が存在するモジュールのみ採用する。WF段階で 架空の補足カード化／KPI展開／案内文 を加えない。情報量が少ないページはそれ自体がKMの開示判断の結果であり、勝手に盛らない。ダミー本文プレフィックス「（文章はダミーです）」は付けてよいが、本文内容は既存原文を保持。
画像「+」拡大ボタン
単独画像PH（グラフ／体制図／模式図／キービジュアル／写真ギャラリー等）にのみ付与する。カード内画像（.activity-card-img 等）には 絶対に付けない。
アコーディオン初期状態
すべて <details open> で 初期展開。WFでは閉じた状態を見せない。階層に応じて .wf-acc-h3（H3配下：グレー帯）／.wf-acc-h4（H4配下：上部横ライン＋白）を使い分け。
ボタンリンク（7-2 / 7-10）の色
白背景＋黒文字＋黒ボーダーのみ。黒背景ボタンは廃止。.is-primary は外枠を2pxに強調するバリアントで、色反転ではない。
トピックエリア（6-1）の形式
左上タグ＋全周ボーダー型で統一。左縦棒型／オーバーレイアイコン型は廃止。タグ文言はページ内容を表す英字短文に（POINT / MATERIALITY KPI / CARBON MINUS 2025 等）。
アンカーナビ（13-2）の設置可否
L3／L4の縦長ページに設置可。ただし 199 ESGデータ／096 アーカイブ／105 ニュース一覧 はユーザー判断で 凍結（非設置）。一覧系・短尺ページはナビ不要との判断。
取り組みカード（7-3）のタグ
カテゴリタグは ジャンル別に具体記載（例：「省エネ」「再エネ」「事業所」）。「取り組み」など一律タグはNG。
テーブル最小行数
データ表は 3行以上を表示（catalog v3.1 ルール）。ニュース行は 4行以上。少なすぎる表は11リストへの置換を検討。
用語モーダル（9-2）の配置
.term-trigger（？マーク丸＋用語名）を ブロック下部の .term-note に配置。本文インライン埋込は避ける。

VIEWPOINT 6 — Anti-patterns

アンチパターン集：やってはいけない組合せ・差し戻し履歴

過去に作って差し戻された／意図的に廃止した構成。クライアントから「これは？」と聞かれた時の即答ストック。

NG

左縦棒型 callout-box（旧 .callout-box の border-left: 3px solid）

H3 .wf-sub-title の左4px縦棒と方向が重複し、視覚的に区別がつかない。

OK

左上タグ＋全周ボーダー型（現行 .callout-box）

タグで「ここがトピック」を即認識でき、H3との方向重複もない。

NG

アコーディオン内のH3アコーディオン入れ子

階層関係が崩れ、ユーザーが構造を見失う。H3配下の長文はH4見出し＋直接表示に。

OK

H3アコーディオン内にH4直接見出し

またはH3直接表示＋H4アコーディオン（.wf-acc-h4）。

NG

取り組みカードに「取り組み」一律タグ

カテゴリが識別できず、カードの並列価値が消える。

OK

ジャンル別の具体タグ（「省エネ」「再エネ」「事業所」等）

カテゴリで識別でき、ユーザーが選んで読める。

NG

カード内画像への「+」拡大ボタン

カード遷移と画像拡大が干渉。クリック誤認の元。

OK

単独画像PHにのみ「+」を付ける

グラフ・体制図・模式図などの単独画像のみ拡大可。
