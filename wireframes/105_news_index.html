設計ポイント ― トピックス一覧（v1.3：得意先FB #231反映）

v1.3 今回の修正サマリー（FB #231）
FB1 WF管理No.を「#105」→「#231（旧 #105）」に変更。新サイト仕様書のページNo.に統一。
FB2 記事タイトルを既存サイト原文で完全踏襲。データソースは /topics/topics_{年}.xml（既存サイトのAngularJSデータ）を取得し、<attribute> に csr を含む記事のみフィルタ（=サステナビリティページ実表示と同条件）。全11年度・216件の記事タイトル・日付を完全踏襲。
FB3 年度チップを「2026〜2015（12年度）」→「2026〜2016（11年度）」に修正。2015年以前は既存サイトに対応XMLが存在しない（HTTP 404）ため除外。
FB4 年度チップをJS駆動でインタラクティブ化。クリックで対応年度の記事リストに即時切替。WF段階で全年度の収録内容を実機同様に確認可能。
FB5
(v1.3追加) Bエリア冒頭の「2025年」年度ラベルおよび「全 N 件」件数表記を削除。年度はチップ側のアクティブ状態で識別可能・件数は冗長との得意先指示。
HERO ページタイトル：シンプルなH1のみ
設計意図 既存ページの「2025年トピックス一覧」のシンプルなH1構成をベースとしつつ、年度はフィルタで切替できるためH1は「トピックス一覧」に統一。装飾的なkicker・サマリカードを排し、本来のページ目的（一覧の閲覧）に集中させる構成。
A 年度フィルタ：11年度chip（2026〜2016）
機能 chipクリックで該当年度の記事リストに切替（JS駆動）。デフォルト＝2025年12件表示。アクティブ年度はchipの黒塗り表示で識別。
記事件数
（既存サイト実数） 2026=1 / 2025=12 / 2024=16 / 2023=25 / 2022=24 / 2021=36 / 2020=35 / 2019=23 / 2018=16 / 2017=13 / 2016=15　／　合計216件
B トピックスリスト：日付＋タイトルのシンプル2列
設計意図 既存ページの「日付＋タイトル」のシンプルなテーブル構成を踏襲。日付列・タイトル列の2要素のみのフラット構造で視線を一直線に導く。ホバー時の背景反転で遷移操作のフィードバックを補強。
原文厳守 記事タイトル・日付は topics_{年}.xml の原文（全角／半角空白も含む）をそのまま反映。架空タイトルなし。
B-2 ページネーション
設計意図 現状の表示件数（年度最大36件）であれば1ページで十分のため、UIは実装イメージとして1ページ（前へ／1／次へ）の状態で示す。年度内件数が多くなる場合の拡張余地を確保。
